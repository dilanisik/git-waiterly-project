# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| Latest (main branch) | ✅ |
| Older versions       | ❌ |

We recommend always using the latest version of the project to benefit from security fixes.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

- 📧 Email: -  
- 🐙 GitHub: Open a private security advisory (preferred)

### What to include in your report:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if possible)

### Response Time:
- Initial response: within 48 hours
- Status update: within 3–5 days

### Disclosure Policy:
- Please do not publicly disclose the vulnerability until it has been reviewed and resolved.
- Once fixed, we may publicly acknowledge the issue.

---

## Security Best Practices

- Do not commit sensitive information (API keys, passwords)
- Use environment variables for secrets
- Keep dependencies up to date
